class openmodelica_microgrid_gym.env.ModelicaEnv(time_step=0.0001, time_start=0, reward_fun=<function ModelicaEnv.<lambda>>, log_level=30, solver_method='LSODA', max_episode_steps=200, model_params=None, model_input=None, model_output=None, model_path='../fmu/grid.network.fmu', viz_mode='episode', viz_cols=None, history=<openmodelica_microgrid_gym.util.recorder.FullHistory object>)[source]¶

Bases: gym.core.Env

OpenAI gym Environment encapsulating an FMU model.

Initialize the Environment. The environment can only be used after reset() is called.

Parameters

  • time_step (float) – step size of the simulation in seconds

  • time_start (float) – offset of the time in seconds

  • reward_fun (Callable[[List[str], ndarray], float]) – The function receives as a list of variable names and a np.ndarray of the values of the current observation. The separation is mainly for performance reasons, such that the resolution of data indices can be cached. It must return the reward of this timestep as float. It should return np.nan or -np.inf or None in case of a failiure. It should have no side-effects

  • log_level (int) – logging granularity. see logging in stdlib

  • solver_method (str) – solver of the scipy.integrate.solve_ivp function

  • max_episode_steps (Optional[int]) –

    maximum number of episode steps. The end time of the episode is calculated by the time resolution and the number of steps.

    If set to None, the environment will never finish because of step sizes, but it might still stop because of system failiure (-inf reward)

  • model_params (Optional[Dict[str, Union[Callable[[float], float], float]]]) –

    parameters of the FMU.

    dictionary of variable names and scalars or callables. If a callable is provided it is called every time step with the current time. This callable must return a float that is passed to the fmu.

  • model_input (Optional[Sequence[str]]) – list of strings. Each string representing a FMU input variable.

  • model_output (Union[dict, Sequence[str], None]) –

    nested dictionaries containing nested lists of strings. The keys of the nested dictionaries will be flattened down and appended to their children and finally prepended to the strings in the nested lists. The strings final strings represent variables from the FMU and the nesting of the lists conveys structure used in the visualisation

    >>> {'inverter': {'condensator': ['i', 'v']}}

    results in

    >>> ['inverter.condensator.i', 'inverter.condensator.v']

  • model_path (str) – Path to the FMU

  • viz_mode (Optional[str]) –

    specifies how and if to render

    • ’episode’: render after the episode is finished

    • ’step’: render after each time step

    • None: disable visualization

  • viz_cols (Union[str, List[Union[str, PlotTmpl]], None]) –

    enables specific columns while plotting - None: all columns will be used for vizualization (default) - string: will be interpret as regex. all fully matched columns names will be enabled - list of strings: Each string might be a unix-shell style wildcard like “*.i”

    to match all data series ending with “.i”.

    • list of PlotTmpl: Each template will result in a plot

  • history (EmptyHistory) – history to store observations and measurement (from the agent) after each step

class openmodelica_microgrid_gym.env.PlotTmpl(variables, callback=None, **kwargs)[source]¶

Bases: object

Provides an iterable of variables and plot parameters like (‘induction1’, {‘color’:’green’, ‘style’: ‘–’}). It contains logic to automatically match up the variables and provided kwargs to allow for a simple syntax.

e.g. when called with [[‘a’,’b’],[‘c’,’d’]] and style = [[‘.’, None],’–’] it will detect the grouping and apply the dotted style to ‘a’ and the dashed style to ‘c’ and ‘d’

Parameters

  • vars – nested list of strings. Each string represents a variable of the FMU or a measurement that should be plotted by the environment.

  • callback (Optional[Callable[[Figure], None]]) – if provided, it is executed after the plot is finished. Will get the generated figure as parameter to allow further modifications.

  • kwargs – those arguments are merged (see omg.util.flatten_together) with the variables and than provided to the pd.DataFrame.plot(·) function

callback(fig)[source]¶

Will be called in the ModelicaEnv.render() once all plotting is finished. This function enables the user to specify more modifications to apply to the figure. The function will call the callable passed in the constructor. Additionally the figure is plotted by this function.

Parameters

fig (Figure) – Finished figure that is one might want to modify.

Returns
